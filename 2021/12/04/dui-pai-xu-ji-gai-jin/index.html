堆排序常用代码： 

package class01;

/**
 * Creat by zhangzhiyuan
 * Date 2021/12/4
 * Time 17:04
 * 堆排序，假设数组长度N，时间复杂度为nlogn（建堆等），空间复杂度O(1)
 */
public class Code09_HeapSort {
    public static void swap(int[] arr,int i, int j){
        int tmp=arr[i];
        arr[i]=arr[j];
        arr[j]=tmp;
    }
    //向上
    public static void heapInsert(int[] arr,int index) {
        while (arr[index] > arr[(index-1)/2]){
            //子位置比父位置的数大，就进行交换
            swap(arr,index,(index-1)/2);
            index=(index-1)/2;   //指针再移动到父位置，准备继续寻找
        }
    }
    //某个数在index位置，判断能否往下移动
    public static void heapify(int[] arr, int index, int heapSize){
        int left=index*2+1; //左孩子
        while (left<heapSize){  //下方还有孩子的时候
            //两个孩子中，选择出大的那个和父交换
            int largest= ((left+1 < heapSize) && (arr[left+1] > arr[left])) ? left+1 : left;  //把孩子中较大的下标给largest
            largest=arr[largest]>arr[index]?largest:index; //父亲和孩子之间，谁的值大，把下标给largest
            if (largest == index){
                break;
            }
            swap(arr,largest,index);
            index=largest;
            left=index*2+1;
        }
    }
    //heap操作添加一个数据，调整级别是logn级别的
    //堆排序
    public static void heap(int[] arr) {
        //调整大根堆的操作~逐步增加数据，从0个开始增加数据，相当于逐步插入数据。
        //删除堆顶，相当于把最后一个和堆顶交换，然后heapsize--，然后调整，这相当于删除了一个元素。
        if (arr==null || arr.length<2){
            return;
        }
		//方法1，常规思路，下面有优化
        for (int i = 0; i < arr.length; i++) {
            heapInsert(arr,i);
        }
        int heapSize=arr.length;
        swap(arr,0,--heapSize);
        while (heapSize > 0) {
            heapify(arr,0,heapSize);
            swap(arr,0,--heapSize);
        }
    }
}

思考：当用户给我一个数组，可以把他默认为一个堆，然后从下往上进行heapfy调整。逐渐形成一个个小堆，然后上面进行一个heapfy，最后构成整个堆。
证明：N个数组成的树，叶子节点大约有N/2个，从他们开始向下进行调整时，复杂度为O(1),倒数第二层节点为N/4个，向下调整复杂度为2，累加之后整体复杂度为T(N)=N/2 *1 + N/4 *2 + N/8 *3 +...，2Tn-Tn，错位相减，估计后为时间复杂度为O(n)

  • 堆结构永远比堆排序重要。
  • 优先级队列就是堆，不是队列。认为堆顶是优先级最大的
    例题：有一个几乎有序的数组，几乎有序是指：一个数移动到它应该所在的位置，距离不超过k。并且k相对数组长度来说较小。
    解答：如果k为6，那么准备一个长度为7的小根堆，然后堆顶就是最小值（因为7位置后面的距离0位置超过6，故不可能为最大值），然后从第8个开始，挨个放到小根堆中，然后选出堆顶，这就是第二小的，然后再拿一个放到小根堆中……这样复杂度为O(N*logK)
    package class01;
import java.util.PriorityQueue;
/**
 * Creat by zhangzhiyuan
 * Date 2021/12/4
 * Time 22:56
 * 建小根堆，给几乎有序的数据进行排序，距离不超过K
 */
public class Code10_SortArrayDistanceLessK {
    public void sortArrayDistanceLessK(int[] arr,int k) {
        //默认小根堆
        PriorityQueue<Integer> heap = new PriorityQueue<>();
        int index=0;
        //建一个小堆，前k个数放进去
        for ( ;index <= Math.min(arr.length,k); index++) {
            heap.add(arr[index]);
        }
        int i=0;
        for ( ;index<arr.length ; i++,index++) {
            heap.add(arr[index]);
            arr[i]=heap.poll();
        }
        while (!heap.isEmpty()){
            arr[i++]=heap.poll();
        }
    }

    public static void main(String[] args) {
        //优先级队列，底层就是堆,可以直接用其来建堆，并且建堆的复杂度都是O（N），扩容的复杂度为（均摊）O(logn)，不影响整体表现。
        PriorityQueue<Integer> heap = new PriorityQueue<>();
        heap.add(123); //添加一个元素
        heap.poll();//弹出并移除最大值。
        //有时候必须要手写堆，因为系统堆不能够支持一些操作（修改某个元素值，然后重新调整，系统会从头扫描，不会从那个元素直接操作）
    }
}

引例：输出流的中位数  
* 题意：
一个流不断的吐出数，吐出的过程会询问中位数的值。  

* 解法：
    维护大根堆，里面放的是吐出来偏小的数。
    维护小根堆，里面是偏大的数。
    每次询问就查询两个堆顶即可。
    * 维护策略：
    一个数如果比大根堆（里面是小的数）的堆顶小，那么insert到这个堆。另一个堆同理。
    当两个堆的size相差>1 ，就把size大的堆的堆顶弹出，加入到另一个堆中。
